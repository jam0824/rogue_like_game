# スキル発動までの流れ

## 1. この資料の目的（設定漏れ防止）
* 敵/プレイヤーの「DB参照 -> 武器ランタイム生成 -> スキル発動」までの実行順を、現行コード準拠で固定化する。
* どのIDをどこで解決しているかを明確にし、設定漏れやID不整合を防ぐ。
* 本資料は実装の現状整理が目的であり、将来仕様の提案や差分議論は扱わない。

## 2. 敵：DB参照からスキル発動まで

### 2.1 参照の入口（DB）
* 敵本体: `db/enemy_db/*.json`
* 武器ロードアウト: `db/enemy_weapon_loadout_db/*.json`
* 武器定義: `db/weapon_db/*.json`
* フォーメーション定義: `db/formation_db/*.json`
* スキル定義: `db/skill_db/*.json`
* エフェクト定義: `db/effect_db/*.json`
* AIプロファイル: `db/enemy_ai_profile_db/*.json`

### 2.2 実行時の解決順（main.js）
1. `refreshEnemyResources()`  
   `enemy_db` + `enemy_ai_profile_db` + `enemy_weapon_loadout_db` を読み込み、`enemyDefinitions` / `enemyAiProfilesById` / `enemyWeaponLoadoutsById` を作る。
2. `refreshWeaponResources()`  
   `weapon_db` + `formation_db` を読み込み、`weaponDefinitionsById` / `formationDefinitionsById` を作る。
3. `refreshSkillResources()` / `refreshEffectResources()`  
   `skillDefinitionsById` / `effectDefinitionsById` を作る。
4. `buildEnemyAttackProfilesByDbId()`  
   敵ごとに `ai_profile_id` と `weapon_loadout_id` を解決し、敵攻撃プロファイルを組み立てる。
5. `createEnemies(...)`  
   攻撃プロファイルから敵武器ランタイム（`attack.weapons[]`）を生成する。
6. フレーム更新 `stepSimulation()` 内  
   `updateEnemyAttacks(...)` で攻撃phase進行 -> start/hitイベント作成 -> `updateEnemySkillChainCombat(...)` でスキル発動。

### 2.3 ID解決の対応表（敵）
* `enemy_db.weapon_loadout_id`
  * 参照先: `enemy_weapon_loadout_db.id`
* `enemy_weapon_loadout_db.weapons[].weapon_def_id`
  * 参照先: `weapon_db.id`
* `enemy_weapon_loadout_db.weapons[].formation_id`（省略可）
  * 参照先: `formation_db.id`
* `enemy_weapon_loadout_db.weapons[].skills[].id`
  * 参照先: `skill_db.id`
* `weapon_db.formation_id`
  * `enemy_weapon_loadout_db.weapons[].formation_id` が未指定/無効時の既定値として利用
* `weapon_db.skills[].id`
  * ランタイム側スキルが未解決のときのフォールバック参照元
* `skill_db.params.projectile.sprite_effect_id` / `skill_db.params.aoe.sprite_effect_id`
  * 参照先: `effect_db.id`

### 2.4 実行上の優先順位（敵）
* フォーメーションID解決順:
  * `loadout.weapons[].formation_id` -> `weapon_db.formation_id` -> 読み込み済み先頭formation
* スキル解決順:
  * runtime skill（enemy loadout由来の `skillInstances`）を優先
  * runtime側が未解決の場合のみ `weapon_db.skills` をフォールバック

### 2.5 stopフォーメーションの扱い（敵）
* `formation.type = "stop"` は敵中心追従のskill専用武器で使う。
* 現行実装では、`buildEnemyAttackProfilesByDbId()` で `type: stop` を受理する。
* stop武器は次を使って「接触ダメージなし/不可視固定」にできる。
  * `supported = false`（軌道演算なし、敵座標スナップ、直接接触ダメージ対象外）
  * `forceHidden = true`（攻撃phase中でも表示しない）
  * `formation.params.weapon_visible` が `true` でない限り stop は不可視運用
* ただしスキルチェーン自体は `weaponStartEvents` / `weaponHitEvents` をトリガーに通常どおり発火する。

### 2.6 enemyスキル発火のイベント経路
1. `updateEnemyAttacks(...)` が攻撃phaseを進め、`attackCycle` を加算する。
2. `buildEnemyWeaponStartEvents(...)` が start event を作る。
3. `buildEnemyWeaponHitEvents(...)` が hit event を作る（`targetType=player`）。
4. `updateEnemySkillChainCombat(...)` が event を受け、`start_spawn_timing` に応じて先頭Attackをキュー投入する。
5. `attack_kind=projectile` なら projectile runtime を生成し、移動と衝突判定を更新する。
6. 命中時 `applyAttackHitToPlayer(...)` で `kind=damage`, `targetType=player`, `sourceType=skill` のイベントを出す。

## 3. プレイヤー：DB/セーブ参照からスキル発動まで

### 3.1 起点の違い（重要）
* プレイヤー見た目定義は `db/player_db/player_01.json`。
* ただし武器/スキルの起点は `player_db` ではなく `player_state`（セーブデータ）。
* 実際の起点は `player_state.run.equipped_weapons[]`。

### 3.2 参照の入口（プレイヤー）
* 見た目: `db/player_db/player_01.json`
* 武器/フォーメーション/スキル: `db/weapon_db/*.json` / `db/formation_db/*.json` / `db/skill_db/*.json`
* 実行時武器インスタンス: `player_state.run.equipped_weapons[].weapon`
  * `weapon_def_id`
  * `formation_id`
  * `skills[]`

### 3.3 実行時の解決順（main.js + playerStateStore.js）
1. `loadDefaultPlayerDefinition()` で `player_db/player_01.json` を読み込み（見た目専用）。
2. `loadPlayerStateFromStorage(...)` で `player_state` を読み込み、`run.equipped_weapons` を正規化。
3. `buildWeaponDefinitionsFromPlayerState(...)` で `weapon_def_id` をキーに WeaponDef を解決し、instance差分（`formation_id`, `skills`, `weapon_plus` など）をマージ。
4. `createPlayerWeapons(...)` で武器ランタイムを作る。
5. ランタイムへ `skillInstances` を設定する（`sourceDefinition.skills` 由来）。
6. `updateWeaponsAndCombat(...)` で武器通常攻撃を更新し、`weaponStartEvents` / `weaponHitEvents` を作る。
7. `updateSkillChainCombat(...)` でプレイヤースキルチェーンを更新し、projectile/aoeとskill damage eventを生成する。

### 3.4 プレイヤー側の注意点
* 現行の通常武器攻撃ランタイム `updateWeaponsAndCombat(...)` は、`formation.type === "circle"` の武器のみ直接攻撃更新を行う。
* `hit_num` と `pierce_count` は2系統ある。
  * 武器直接攻撃: `weapon_db.hit_num` / `weapon_db.pierce_count`
  * スキル攻撃: `skill_db.params.hit.hit_num` / `skill_db.params.hit.pierce_count`

## 4. 設定者チェックリスト（敵/プレイヤー）

### 4.1 敵1体を追加してスキルを撃たせるとき
* [ ] `db/enemy_db/*.json` に `ai_profile_id` と `weapon_loadout_id` を設定した
* [ ] `db/enemy_ai_profile_db/*.json` に攻撃phase/aim/visibility（`weapon_aim_mode`, `weapon_visibility_mode` など）を設定した
* [ ] `db/enemy_weapon_loadout_db/*.json` に `weapon_def_id`, `formation_id`, `skills[].id` を設定した
* [ ] `db/weapon_db/*.json` に `id`, `formation_id`, `skills`, `base_damage`, `attack_cooldown_sec` を設定した
* [ ] `db/formation_db/*.json` に `type` と `params`（stopなら `weapon_visible`）を設定した
* [ ] `db/skill_db/*.json` に `attack_kind`, `start_spawn_timing`, `hit.hit_num`, `hit.pierce_count`, `projectile.sprite_effect_id` を設定した
* [ ] projectile/aoeを見せるなら `db/effect_db/*.json` の effect ID を作成した

### 4.2 プレイヤー側で武器/スキルを成立させるとき
* [ ] `player_state.run.equipped_weapons[].weapon.weapon_def_id` を設定した
* [ ] `player_state.run.equipped_weapons[].weapon.formation_id` を設定した
* [ ] `player_state.run.equipped_weapons[].weapon.skills[]` を設定した
* [ ] 参照先の `weapon_db` / `formation_db` / `skill_db` に対応IDが存在する

## 5. 症状別の確認ポイント

| 症状 | まず確認する箇所 |
|---|---|
| スキルダメージは出るが弾エフェクトが見えない | `skill.params.projectile.sprite_effect_id` -> `effect_db.id` の一致、effectアセット読込、`life_sec` が短すぎないか、命中直後に消えていないか |
| 敵が攻撃しているのにskillが出ない | `start_spawn_timing` と event種別の整合（`start` は `weaponStartEvents`、`hit` は `weaponHitEvents`） |
| stop武器で接触ダメージが出る/出ない | `supported` と `forceHidden` の意図確認（skill専用なら `supported=false` + `forceHidden=true`） |
| `hit_num` の効き方が期待と違う | 武器直接攻撃（`weapon_db.hit_num`）か、skill攻撃（`skill_db.params.hit.hit_num`）かを切り分ける |
| `pierce_count` の効き方が期待と違う | 武器直接攻撃（`weapon_db.pierce_count`）か、skill projectile/aoe（`skill_db.params.hit.pierce_count`）かを切り分ける |
| DBを追加したのに読まれない | 対象ローダーのディレクトリ列挙失敗時fallbackにIDが入っているか、ID重複/typoがないか |

## 6. BrownMushroom + bite の具体トレース

### 6.1 参照ファイルとID
* `db/enemy_db/BrownMushroom_01.json`
  * `weapon_loadout_id = enemy_loadout_bite_01`
  * `ai_profile_id = ai_profile_chaser_v1`
* `db/enemy_weapon_loadout_db/enemy_loadout_bite_01.json`
  * `weapons[0].weapon_def_id = weapon_enemy_bite_01`
  * `weapons[0].formation_id = formation_id_stop01`
  * `weapons[0].skills[0].id = skill_id_bite_01`
* `db/weapon_db/weapon_enemy_bite_01.json`
  * `formation_id = formation_id_stop01`
  * `skills[0].id = skill_id_bite_01`
* `db/formation_db/formation_stop_01.json`
  * `id = formation_id_stop01`
  * `type = stop`
  * `params.weapon_visible = false`
* `db/skill_db/skill_id_bite_01.json`
  * `params.attack_kind = projectile`
  * `params.start_spawn_timing = start`
  * `params.hit.pierce_count = 99`
  * `params.projectile.sprite_effect_id = effect_id_bite_01`
* `db/effect_db/effect_id_bite_01.json`
  * bite projectile表示に使うeffect定義

### 6.2 発火条件と最終イベント
* 敵攻撃phaseが `ATTACK` に入って `attackCycle` が進む。
* `buildEnemyWeaponStartEvents(...)` が該当weaponのstart eventを作る。
* `updateEnemySkillChainCombat(...)` が `start_spawn_timing = start` の先頭Attackとして `skill_id_bite_01` を起動する。
* bite projectileがプレイヤーに当たると、`kind=damage`, `targetType=player`, `sourceType=skill`, `skillId=skill_id_bite_01` のイベントが発生する。
* `applyPlayerHpDamage` が有効ならHPが減少し、無効でもdamage event自体は出る（プレビュー用途）。
