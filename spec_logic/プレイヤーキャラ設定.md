# プレイヤーキャラ設定
* プレイヤーキャラの設定はdb/player_dbに保存されている
* デフォルトのプレイヤー設定はplayer_01.jsonである
* 壁は通り抜けることができない。
* 敵の攻撃を受けた際の当たり判定はキャラクターと同サイズである。
* 敵の攻撃を受けた際の当たり判定はスキルによって小さくなる場合がある。

# プレイヤーキャラのチップ設定について
* 画像は横1列にアニメーションが並んでいる
* 1コマはwidth, heightで定義されている。
* プレイヤーのアニメーションにのみ適用されるfpsが定義されている
* player_png_facing_directionが、画像内のキャラクターのデフォルトの向きである
* プレイヤーの歩行モーションは横向きのみである
* 横に移動しているときは、walk_png_file_pathのアニメーションが再生される
* 最後のコマまで再生したら最初のコマに戻る
* 上下に移動するときも横向きのアニメーションのままである
* idle_png_file_pathが動いていないときのidleアニメーションである
* HPが0になったときは death_png_file_pathのアニメーションである
    * ただしHPが0になった時は最後のアニメーションのコマでストップする
    

# プレイヤーの移動について
* プレイヤーはマウスクリックがあった場所に一直線に移動する
* 移動途中に壁があった場合、そこで止まる
* マウスを押している間、マウスを追従する
* 上、下へ移動しているときは、数ピクセルの左右への移動で左右のアニメーションを切り替えない。マージンを持たせる（アニメーションが頻繁に切り替わらないための措置）
* マウス話したらその時点で移動をやめる
