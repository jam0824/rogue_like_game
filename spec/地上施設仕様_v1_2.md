# 地上施設

## 保管庫

### 目的
- **ラン内の手持ち（`run.inventory[]` / 装備）** と、**永続保管（`base.stash`）** の間でアイテムを整理する施設。
- **売却**（換金）と、**容量拡張**（保管庫／手持ち袋）を行える。

### ランとの関係（ライフサイクル）
- 地上にいる時のみ、`base.stash` ↔ `run.inventory` の出し入れが可能。
- **ポータル帰還後も run は継続**する（`in_run: true` 継続）ため、
  - 地上で整理してから同じランで再突入できる。
  - いったん **stash に預けた物は、以降そのランで死亡しても残る**（run ロストの影響を受けない）。
- **死亡**すると `run.inventory / run.equipped_*` は全ロスト（護符がある場合の例外は別仕様）し、`base.stash` は残る。

> 参照：`ステータス仕様_v1_4_3.md` の `player_state.json`（`base.stash` / `run.inventory`）。

### 取り扱い対象
- v1：消耗品（item）／武器（weapon）／防具（armor）／アクセサリー（accessory）
- （任意/将来）スキルチップ単体所持を実装する場合は `type:"chip"` を追加して同様に扱う（CHIPタブと連動）。

### 容量（スロット）ルール
- `base.stash.capacity` が **保管庫スロット上限**。
- `stash.items[]` の **要素1つ = 1スロット**。
- `type:"item"` はスタック可能。
  - `count` は **ItemDef.max_stack を超えない**（超える場合は複数スタックに分割して格納）。
  - 預け入れ時は可能な限り **自動で同種スタックに合流**（上限まで埋める）。
- `type:"weapon" / "armor" / "accessory"` は非スタック（常に1枠）。
- 空きスロットが足りない場合は預け入れ不可（UIでブロック＋トースト）。

### 未鑑定・呪いの扱い（v1）
- 未鑑定装備は `identified:false` として保持できる。
- UI表示
  - 未鑑定：アイコンに「？」オーバーレイ、名称は伏せる（例：`???`）。
  - （呪いを実装する場合）呪い：警告アイコンを重ねる。

### UI（保管庫ウィンドウ）
- 2カラム構成（モーダル）
  - 左：**手持ち（`run.inventory`）**（最大 `inventory_slot_max` 枠）
  - 右：**保管庫（`base.stash`）**（`capacity` 枠）
- 上部タブ（推奨）
  - `ALL / ITEM / WEAPON / ARMOR / ACCESSORY`（将来 `CHIP`）
- 右側または下部に **選択中詳細**
  - 名前、説明、レア、+値、（装備なら）主要パラメータ、売却価格、未鑑定表示など

### 操作（入出庫）
- 方式（どちらか or 両対応）
  - スロット選択 → ボタンで移動（`預ける` / `引き出す`）
  - ドラッグ&ドロップ（対応プラットフォームのみ）
- `item`（スタック）操作
  - `全量移動`（スタックごと）
  - `分割移動`（指定個数）
- 便利機能（推奨）
  - `自動整頓`（同種合流→ソート）
  - `並び替え`（入手順／名前／レア度／+値）
  - `ロック`（誤売却・誤分解防止）

### 売却（換金）
- 保管庫ウィンドウ内で売却できる。
  - 対象：`run.inventory` 側／`base.stash` 側どちらからでも可（UIで選べる）
  - 売却後は即消滅し、`wallet.gold` に加算
- 売却モード（推奨）
  - `売却モードON` → 複数選択 → `合計金額` 表示 → `売却` 確認
- 安全策（推奨）
  - `locked:true` のものは売却不可
  - `rare/legendary` や `+` が高いものは追加確認
  - 未鑑定装備は「未鑑定価格」で売却（鑑定後の方が高い）などの差別化も可

> 売却価格テーブルはエコノミー仕様（別紙）に切り出し推奨。v1は仮固定でもよい。

### アップグレード
- **保管庫容量拡張**
  - `base.stash.capacity` を増やす
- **手持ち袋容量拡張**（推奨：保管庫で提供）
  - `run.inventory` の最大枠（`inventory_slot_max`）を増やす
  - ※保存先は `base.unlocks.inventory_slot_max` 等に保持する（既存unlockに追記）

> 拡張コストは段階式（テーブル）か、係数式（`base_cost * growth^level`）で管理し、バランス調整しやすくする。

### データ（セーブ）
- `base.stash`
  - `capacity:int`
  - `items[]`: `type` で `item/weapon/armor/accessory` を混在（詳細はステータス仕様参照）
- （任意）誤操作防止や並び替え品質のため、以下のフィールド追加を推奨
  - `uid`（インスタンス識別子）
  - `locked`（売却/分解禁止）
  - `acquired_at`（入手順ソート用）

## ショップ
* 使い切りアイテムを売っている
* 善良度で値段が安くなる
* 悪名度で値段が高くなる

## 鑑定屋
* 地上に持ち帰った未鑑定のアイテムをまとめて鑑定することができる

## ゲート
* 10Fのボスを倒した後に解放
* これまで帰還したダンジョンの階にワープすることができる

## 冒険者ギルド
* ギルドではクエストを受けることができる
* クエストの種類
    * 指定アイテムを持ってくる
    * 指定の敵を倒してその時ドロップする証拠アイテムを持ってくる
    * 探し人でダンジョン内に倒れている人を助ける
    * 賞金首の討伐
* クエストではお金がもらえる
* クエストをこなすと冒険者ランクが上がる
* 受けられるクエストは冒険者ランクで決まっている
* 冒険者ランクが上がった時にもらえるアイテムがある。上位ほどレアやユニークな武器やスキルが手に入る
* クエストでは賞金首の討伐もあるが、このときは倒しても悪名度が上がらない。

## 精錬所
* 同じ装備(武器、防具、アクセサリー)を合成することができる施設。
* 合成後、ベースとなる装備は+1される。最大+99まで。
* 合成後、ベースとなる装備にあるスキルは+1される。ベースにないスキルはベース側に空きスロットがあるならつくが、空きスロットが足りないスキルは破棄される
* 一律で同じ金額がかかる

## 防具仕立て屋
* 防具の耐久値を回復させることができる
* 防御力に応じて金額が変わる

## 分解屋
* 不要な装備/アイテムを分解して、合成素材（欠片・粉・コア等）に変換できる
* 分解結果はレア度/強化+で増減する（テーブル管理推奨）
* `locked:true` のものは分解不可

## 闇市
* 悪名度が高いと解放される
* 死んだときにアイテムを一つだけ守る護符など、珍しいアイテムを買える
* その他色んなヤバいものが変える
* ただしめちゃくちゃ高い
* **裏解呪**：呪い解除（神殿より割高だが条件が緩い等、差別化してもよい）

## 神殿
* 善良度が高いと解放される
* 次のrun内でのみ有効なバフをかけることができる
* お金もかかる
* **お祓い（解呪）**：呪い解除（善良寄りの導線）

## 魂の鍛錬
* 永続的にステータスをあげることができる
* ダンジョン内で拾える魂の精錬用アイテム(legend or drop)とお金を使用する

## 図書館
* コレクション要素。
* これまで遭遇した敵が見れる。全体の何%と遭遇したかがわかる
* これまで取得した、武器、防具、アクセサリー、アイテムが分かる。全体の何％を見たかわかる
* これまで取得したスキルが分かる。全体の何%を見たかわかる

## 訓練場
* ダメージや発動挙動を確認するためのテスト用エリア
* 報酬なし、いつでも退出可能
* 推奨：DPS表示／ヒット数／状態異常付与量の簡易ログを出せる

## モンスター牧場
* テイムしたモンスターをあずけたり引き出せたりする
