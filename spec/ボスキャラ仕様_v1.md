# ボスキャラ概要
* ボスキャラは10,20,30,40,50,60,70,80,90,100階で現れる
* ボスフロアはアリーナ型の大きい部屋1つである
* ボスを倒すまで下へ行く階段は現れない。ボスを倒したら現れる
* ボスを倒したときはlegendaryの宝箱1つ出現させる

# 各ボスキャラ設定

## 10F・オーガ

### コンセプト（10Fの学習テーマ）
- 「予告（テレグラフ）を見て回避 → 硬直に反撃」の基本を覚えさせる節目ボス
- 技は3種のみで、読みやすい代わりに**位置取りを崩す力**（突進）と**場圧**（召喚）でボスらしさを出す

### 基本情報（初期案）
- 種別：大型近接ボス
- 武器：石槌（大きいハンマー）
- Rank：Boss
- 想定ロール：chaser + charger + summoner
- 推奨tags：`["heavy"]`（重量感・よろけにくさ）

> 数値（HP/攻撃/状態耐性など）は「Floor倍率 × Rank倍率 × Tag倍率」を前提に、まずは動く基準で調整していく。

### ボス部屋（アリーナ）
- 広めの1部屋（回避余地を確保）
- 柱 2〜4 本（突進の避難所／壁衝突の誘導に使う）
- 罠は基本無し（10Fではボスの読み合いに集中）
- 10F開始時のプレイヤー出現位置は、ボス位置から **マンハッタン距離 10 タイル以上** を確保する（`|dx| + |dy| >= 10`）

### フェーズ設計（同じ3技の"強化"で差を出す）
- Phase1：HP 100%〜60%
  - 基本形（読みやすい）
- Phase2：HP 60%〜30%
  - 突進に軽いフェイント（微補正）を追加
  - 召喚の回転を少し上げる（CT短縮 or 1体増）
- Phase3：HP 30%〜0%
  - 召喚数を増やす（上限に注意）
  - 石槌プレスを2連にする（2発目は小範囲／低ダメでもOK）

## 行動（AI）

### 優先度（ざっくり）
1. **取り巻きが不足**していて召喚CTが明けている → 取り巻き召喚
2. 距離が遠く、突進CTが明けている → 突進
3. 距離が近く、プレスCTが明けている → 石槌プレス
4. それ以外 → 追跡（chaser）

### 連打防止（テンポ調整）
- 各攻撃の後に `recover`（硬直）を必ず入れる
- 「突進→即プレス」など、事故りやすい連携はPhase3以降に限定するか、間に短い間合い調整を挟む

---

## 攻撃パターン

> 表記：`windup（予告） / active（発動） / recover（硬直）`
> いずれも初期値。プレイテストで短縮・延長して調整する。

### 1) 突進（Charger：ライン回避）
- 目的：逃げ続けを咎め、柱・壁を使った回避を覚えさせる
- 発動条件：
  - プレイヤーとの距離が一定以上（目安：5タイル以上）
  - かつ突進CTが明けている
- テレグラフ：赤いライン（幅 0.8〜1.0タイル）**コード側プリミティブ描画（半透明赤）**
- 方向ロック：**windup開始時に方向確定**（基本は追尾しない）

**タイム（初期値）**
- windup：0.7〜0.9s
- active：直線ダッシュ（距離 6〜8タイル）
- recover：0.6s
  - 壁/柱に激突して止まった場合：recover を 1.2s に延長（避けたご褒美）

**ヒット仕様（初期値）**
- 接触時：物理ダメージ + ノックバック
- 10Fの推奨：**プレイヤーに当たったら止まる**（読みやすさ優先）

**フェーズ変化**
- Phase2：windup後半で 10〜15° 程度の微補正（フェイント感）
  - その場合、ラインをわずかに扇形にして「補正の余地」を視覚で伝える

---

### 2) 取り巻き召喚（Summoner：場圧）
- 目的：ボス単体の読み合いに「処理優先度」を足す（密度コントロールの学習）
- 発動条件：
  - 取り巻きの生存数が少ない（例：<3）
  - または召喚CTが明けている
- テレグラフ：咆哮 + 出現地点に小円（0.5s程度）**コード側プリミティブ描画（半透明赤円）**

**タイム（初期値）**
- windup：0.9〜1.1s
- active：取り巻き出現
- recover：0.4〜0.6s（咆哮後の隙）

**召喚数（初期値）**
- Phase1：2体
- Phase2：2〜3体
- Phase3：3〜4体

**召喚位置（初期値）**
- ボス周囲の安全距離外（プレイヤー足元直湧きは避ける）
- もしくは部屋の四隅／柱の裏（テレグラフで視認できる配置）

**召喚体（推奨仕様）**
- 種別：小型近接（swarm/chaser）
- tags：`["summoned","minion"]` 推奨
  - `summoned`：XP=0、ドロップ無し
  - `minion`：HP低め・処理しやすい枠
- 同時存在上限（初期値）：
  - 部屋内最大 8体
  - summoner（オーガ）1体につき最大 6体
- オーガ死亡時：召喚体は即時消滅してよい（テンポ維持）

---

### 3) 石槌プレス（円AoE：置き攻撃）
- 目的：近距離の欲張りを咎め、**避けてから殴る**の基本を作る
- 発動条件：
  - プレイヤーが近距離（目安：3タイル以内）
  - または突進後の追撃として
- テレグラフ：着弾円（半径 1.5〜2.0タイル）**コード側プリミティブ描画（半透明赤円）**
- 位置ロック：**windup開始時のプレイヤー位置に固定**（追尾しない）（スキル仕様の `aoe.target_position="target_locked"` + `position_lock_timing="on_windup_start"` を使用）
- effect表示：`effect_ogre_hammer_press_01` は **ロック済み着弾中心（テレグラフ中心）** に表示する。命中の有無に関係なく active で再生する。

**タイム（初期値）**
- windup：0.9〜1.2s
- active：着弾
- recover：0.7〜0.9s（反撃時間を確保）

**ヒット仕様（初期値）**
- 中心円：大ダメージ
- Phase2以降（任意）：0.25s後に外周リング（小ダメージ）
  - 「早避け→即戻り」を完全否定しない程度の圧に留める

**フェーズ変化**
- Phase3：2連プレス
  - 2発目は範囲を少し小さく／ダメージ控えめでもOK（テンポ重視）

---

## 報酬・進行
- 撃破時：legendary宝箱1つを出現
- 撃破時：下り階段を出現（それまでは出現しない）
- 10F撃破は節目なので、演出（咆哮停止→崩れ落ち→宝箱湧き）を強めにして達成感を出す

## 召喚体（OgreMinion）推奨DB定義

オーガが召喚する取り巻き（OgreMinion_01）の推奨パラメータ。実装者はこれをベースに `enemy_db` / `enemy_ai_profile_db` / `enemy_weapon_loadout_db` を作成する。

### enemy_db（推奨値）
```json
{
  "name_key": "name_OgreMinion",
  "description_key": "description_OgreMinion",

  "type": "walk",
  "walk_png_file_path": "graphic/enemy/OgreMinion/walk_OgreMinion.png",
  "idle_png_file_path": "graphic/enemy/OgreMinion/idle_OgreMinion.png",
  "attack_png_file_path": "graphic/enemy/OgreMinion/attack_OgreMinion.png",
  "death_png_file_path": "graphic/enemy/OgreMinion/death_OgreMinion.png",
  "width": 32,
  "height": 32,
  "fps": 12,
  "png_facing_direction": "right",
  "image_magnification": 1.5,
  "notice_distance": 6,
  "giveup_distance": 999,

  "rank": "normal",
  "role": "swarm",
  "spawn": { "min": 1, "max": 1 },

  "vit": 6,
  "for": 4,
  "agi": 3,
  "pow": 8,
  "tec": 1,
  "arc": 1,

  "tags": ["summoned", "minion"],

  "ai_profile_id": "ai_profile_swarm_v1",
  "weapon_loadout_id": "enemy_loadout_ogre_minion_claw_01"
}
```

### 設計意図
- **tags**: `summoned`（XP=0、ドロップなし）+ `minion`（HP×0.60、攻撃×0.70で処理しやすい）
- **role**: `swarm`（近接群れ。プレイヤーに接近して纏わりつく）
- **ステータス**: 10F相当の Normal 基礎値よりさらに低め。tag の `minion` 倍率でさらに下がる
- **画像**: オーガの小型版（32x32、拡大1.5倍）。既存の敵素材を流用してもよい
- **AI**: 既存の swarm 用プロファイルを使用。専用AIは不要
- **武器**: 近接の爪/噛みつき系で、ボスキャラ仕様の召喚体概要（小型近接）に合わせる

### weapon_loadout（推奨値）
```json
{
  "id": "enemy_loadout_ogre_minion_claw_01",
  "attack_linked": true,
  "weapons": [
    {
      "weapon_def_id": "weapon_enemy_claw_01",
      "rarity": "common",
      "weapon_plus": 0,
      "formation_id": "formation_id_stop01",
      "skills": [
        { "id": "skill_id_bite_01", "plus": 0 }
      ]
    }
  ]
}
```

> 取り巻きは処理しやすさ優先。武器は既存の敵用共通武器（爪/噛みつき）を流用し、新規作成は不要。

## DB参照構造と実行フロー（10Fオーガ）

オーガのDBは複数ファイルに分かれている。混乱を防ぐため、**参照関係**と**実行時の流れ**をここにまとめる。

### DB参照の全体像

下図で `→` は「IDで参照している先」を示す。実装時はこの参照チェーンが正しく繋がっているか確認すること。

```
dungeon_id_10.json                          ← 10Fフロア定義
  └─ enemy_db_ids: ["Ogre"]
       │
       ▼
Ogre.json (enemy_db)                        ← ボス本体（見た目・ステータス）
  │
  ├── ai_profile_id ─────→ ai_profile_boss_ogre_v1.json (enemy_ai_profile_db)
  │                          │
  │                          ├─ phases[]             … HP比率でフェーズ自動切替
  │                          ├─ action_priority[]    … 行動選択の優先順位（上から評価）
  │                          ├─ actions              … 行動ごとの詳細パラメータ
  │                          │   ├─ charge  → weapon_index: 0 ─┐
  │                          │   ├─ press   → weapon_index: 1 ─┤
  │                          │   └─ summon  → weapon_index: 2 ─┤
  │                          └─ summon_rules          … 召喚上限ルール
  │                                                             │
  └── weapon_loadout_id ──→ enemy_loadout_ogre_boss_01.json     │ weapon_index で対応
                             │                                  │
                             ├─ weapons[0] (action_key:"charge")←┘
                             │   └→ weapon_ogre_charge_01 (weapon_db)
                             │       └→ skill_ogre_charge_01 (skill_db)
                             │           └→ effect_ogre_charge_hit_01 (effect_db)
                             │
                             ├─ weapons[1] (action_key:"press")
                             │   └→ weapon_ogre_stone_hammer_01 (weapon_db)
                             │       └→ skill_ogre_hammer_press_01 (skill_db)
                             │           └→ effect_ogre_hammer_press_01 (effect_db)
                             │
                             └─ weapons[2] (action_key:"summon")
                                 └→ weapon_ogre_summon_01 (weapon_db)
                                     └→ skill_ogre_summon_01 (skill_db)
                                         ├→ effect_ogre_summon_cast_01 (effect_db)
                                         └→ enemy_id: "OgreMinion_01" (enemy_db)
                                              ├→ ai_profile_swarm_v1
                                              └→ enemy_loadout_ogre_minion_claw_01
                                                   └→ weapon_enemy_claw_01
```

### 行動1回の実行フロー

オーガが行動を選択してから完了するまでの流れ。全行動（突進・プレス・召喚）で共通。

```
 ┌─────────────────────────────────────────────────────────┐
 │ 1. フェーズ判定（AIプロファイル）                          │
 │    現在HP比率 → phases[] から該当フェーズを決定            │
 │    → 召喚数・微補正角度・プレス連鎖数をフェーズ値で上書き   │
 └──────────────────────┬──────────────────────────────────┘
                        ▼
 ┌─────────────────────────────────────────────────────────┐
 │ 2. 行動選択（AIプロファイル）                              │
 │    action_priority[] を上から順に評価                      │
 │    → 条件（距離・CT・召喚数）が真の最初の行動を採用        │
 │    例: charge の when = "target_distance_gte && cooldown_ready" │
 └──────────────────────┬──────────────────────────────────┘
                        ▼
 ┌─────────────────────────────────────────────────────────┐
 │ 3. 行動パラメータ取得（AIプロファイル → ロードアウト）     │
 │    actions.charge を読む → weapon_index: 0               │
 │    ロードアウトの weapons[0] を取得                       │
 │    → weapon_def_id, skills を確定                        │
 └──────────────────────┬──────────────────────────────────┘
                        ▼
 ┌─────────────────────────────────────────────────────────┐
 │ 4. スキル実行（skill_db の params で挙動が決まる）        │
 │                                                          │
 │  [windup]  テレグラフ表示（telegraph_style でコード描画）  │
 │            方向/位置ロック（direction_lock_timing 等）     │
 │       ↓                                                  │
 │  [active]  突進移動 / AoE着弾 / 召喚体出現               │
 │            ヒット判定 → ダメージ計算 → エフェクト再生      │
 │       ↓                                                  │
 │  [recover] 硬直（プレイヤーの反撃チャンス）                │
 └──────────────────────┬──────────────────────────────────┘
                        ▼
              1. に戻って次の行動を選択
```

### 各行動のDB参照チェーン（早見表）

行動名の右に、その行動で順番にたどるDBファイルとキーフィールドを記載する。

**突進（charge）**
```
ai_profile.actions.charge
  → weapon_index: 0
  → loadout.weapons[0].weapon_def_id = "weapon_ogre_charge_01"
    → weapon_db.skills[0].id = "skill_ogre_charge_01"
      → skill_db.params.charge
          dash_distance_tiles: 7.0    … 突進距離
          speed_tile_per_sec: 12.0    … 突進速度
          telegraph_style: "line_red_translucent"  … テレグラフ（コード描画）
          stop_on_player_hit: true    … 接触で停止
      → skill_db.effect_id_hit = "effect_ogre_charge_hit_01"
          → effect_db で命中/壁衝突エフェクトを再生
```

**石槌プレス（press）**
```
ai_profile.actions.press
  → weapon_index: 1
  → loadout.weapons[1].weapon_def_id = "weapon_ogre_stone_hammer_01"
    → weapon_db.skills[0].id = "skill_ogre_hammer_press_01"
      → skill_db.params.aoe
          target_position: "target_locked"           … windup開始時のプレイヤー位置に固定
          position_lock_timing: "on_windup_start"    … ロックタイミング
          telegraph_style: "circle_red_translucent"  … テレグラフ（コード描画）
          telegraph_radius_tiles: 2.0                … 予告円の半径
          sprite_effect_id: "effect_ogre_hammer_press_01"
            → effect_db で着弾エフェクトを再生（当たり判定もこのサイズ基準）
            → target_locked + on_windup_start のため、表示中心はロック済み座標（テレグラフ中心）に固定
            → 命中の有無に関係なく active 開始時に再生
```

**取り巻き召喚（summon）**
```
ai_profile.actions.summon
  → weapon_index: 2
  → loadout.weapons[2].weapon_def_id = "weapon_ogre_summon_01"
    → weapon_db.skills[0].id = "skill_ogre_summon_01"
      → skill_db.params.summon
          enemy_id: "OgreMinion_01"             … 召喚する敵のenemy_db ID
          count: { min: 2, max: 2 }             … 召喚数（phases[] で上書きされる）
          spawn_style: "boss_ring_outside"       … 出現位置の決め方
          spawn_telegraph_style: "circle_red_translucent"  … 出現地点テレグラフ（コード描画）
          cast_effect_id: "effect_ogre_summon_cast_01"
            → effect_db で咆哮演出を再生
      → OgreMinion_01.json (enemy_db)
          → ai_profile_swarm_v1  … 群れAIで自律行動
          → enemy_loadout_ogre_minion_claw_01  … 爪/噛みつき攻撃
```

### weapon_index とロードアウトの対応

AIプロファイルの `actions.<行動>.weapon_index` は、ロードアウトの `weapons[]` 配列の**インデックス**で武器を指定する。ロードアウト側の `action_key` はドキュメント上の対応確認用。

| action | weapon_index | loadout.weapons[] の action_key | 武器DB | スキルDB |
|---|:---:|---|---|---|
| charge | 0 | `"charge"` | weapon_ogre_charge_01 | skill_ogre_charge_01 |
| press | 1 | `"press"` | weapon_ogre_stone_hammer_01 | skill_ogre_hammer_press_01 |
| summon | 2 | `"summon"` | weapon_ogre_summon_01 | skill_ogre_summon_01 |

### DBファイル一覧（10Fオーガ）

| 区分 | 置き場所 | 主なID例 | 役割 |
|---|---|---|---|
| ボス本体 | `db/enemy_db` | `Ogre` | 見た目、基礎ステ、rank/role/tags、AI/武器ロードアウト参照 |
| AIプロファイル | `db/enemy_ai_profile_db` | `ai_profile_boss_ogre_v1` | フェーズ、優先行動、行動CT、距離条件、召喚上限条件 |
| 武器ロードアウト | `db/enemy_weapon_loadout_db` | `enemy_loadout_ogre_boss_01` | 突進/プレス/召喚で使う武器インスタンス配列 |
| 武器定義 | `db/weapon_db` | `weapon_ogre_charge_01` / `weapon_ogre_stone_hammer_01` / `weapon_ogre_summon_01` | 攻撃周期、基礎ダメ、参照スキル |
| スキル定義 | `db/skill_db` | `skill_ogre_charge_01` / `skill_ogre_hammer_press_01` / `skill_ogre_summon_01` | 行動単位の挙動パラメータ |
| 召喚体 | `db/enemy_db` | `OgreMinion_01` | `summoned` / `minion` タグ付き取り巻き |
| エフェクト定義 | `db/effect_db` | 下記リスト参照 | 着弾・召喚演出の可視化（テレグラフはコード描画） |
| ダンジョン | `db/dungeon_db` | `dungeon_id_10` | 10Fボスフロア定義（`boss_floor: true`） |

## 実装者向け画像作成リスト（effect素材）

以下はオーガ実装で画像担当が先に用意するとよい素材。IDは `effect_db` 側の `id` と揃える。

> 突進ライン予告・召喚地点マーカーのテレグラフは **effect画像ではなくコード側のプリミティブ描画**（半透明赤ライン/円）で表示する（スキル仕様の `telegraph_style` / `spawn_telegraph_style` 参照）。

| effect_id | 想定ファイル名例 | 用途 | 推奨サイズ（px） | loop | 当たり判定の考え方 |
|---|---|---|---:|---|---|
| `effect_ogre_hammer_press_01` | `graphic/effect/effect_ogre_hammer_press_01.png` | 石槌プレス着弾円（中心大ダメ） | 192x192 | false | 円中心基準。半径1.5〜2.0タイル相当 |
| `effect_ogre_hammer_press_ring_01` | `graphic/effect/effect_ogre_hammer_press_ring_01.png` | Phase2以降の外周リング（任意） | 224x224 | false | 外周のみを強調し、中心再被弾を避ける見た目 |
| `effect_ogre_summon_cast_01` | `graphic/effect/effect_ogre_summon_cast_01.png` | 召喚詠唱（咆哮）演出 | 192x192 | false | 当たり判定なし想定（演出用） |
