## 0. スコープ

本書で定義するもの（v1）：
- **アイテム（道具）**：ラン中に所持し、主にインベントリから使用するもの（消耗品／鑑定系など）
- **装備**：武器／防具／アクセサリー（装備スロットに付けるもの）
- **スキル**：武器に装着するスキルチップ（スキル定義・発動ルールの概要）

本書で詳細を定義しないもの：
- 武器軌道（フォーメーション）やスキルチップ詳細パラメータ：武器システム仕様を参照
- 状態異常の付与量・スタック・DoT計算：ステータス仕様を参照
- 宝箱カテゴリ・ドロップ比率：ダンジョン仕様を参照
- UIレイアウト：システムUI仕様を参照

---

## 1. UI・所持枠（要点）

- **簡易アイテムインベントリ（クイックスロット）**：最大 **8枠**
  - 登録したアイテムをクリックで即使用
- **アイテムインベントリウィンドウ**：初期 **最大10個**
  - 地上施設で最大数を拡張できる
  - 選択したアイテムを **USE / DROP** できる
- 多言語対応：表示名・説明は **language_key**（例：`name_key` / `description_key`）で管理する

---

## 2. 用語（データ視点）

- **ItemDef**：アイテム定義（マスターデータ）。IDごとに固定の情報（名前、説明、効果種別、アイコン、最大スタック等）
- **ItemInstance**：所持しているアイテム個体。`item_def_id` と `count`（個数）などを保持
- **装備（Equipment）**：武器／防具／アクセサリー。多くは非スタック（1個＝1枠）
- **スキルチップ（SkillChip）**：武器に装着するスキル。`id` と `plus`（強化+）で表現する

---

## 3. 命名ルール（v1）

カテゴリごとに“語尾（見た目の統一）”を揃える：

- 回復：**草／エリクサー**
- 状態異常ケア：**〜薬**
- バフ：**〜の香**
- 投擲：**〜瓶**
- 鑑定：**〜本／手帳**

## 4. アイテム（道具）一覧（v1 名称確定）

### 4.1 回復（HP回復）

| 名前 | 効果（概要） |
|---|---|
| 薬草（小回復） | HPを小回復 |
| 回復ポーション（中回復） | HPを中回復 |
| エリクサー | HPを大回復（※追加効果は必要なら後で付与） |

> 回復量の数値（固定値/最大HP割合）は v1実装で決める（本書では名称と役割を確定）。

---

### 4.2 状態異常ケア（解除・軽減）

| 名前 | 対象（対応する状態異常） | 効果（概要） |
|---|---|---|
| 止血薬 | 出血 | 出血を解除（またはスタックを大きく減少） |
| 解毒薬 | 中毒 | 中毒を解除（またはスタックを大きく減少） |
| 消化薬 | 燃焼 | 燃焼を解除（またはスタックを大きく減少） |
| 温熱薬 | 冷却（→凍結の予防） | 冷却を解除/軽減（凍結の昇格を防ぐ） |
| 絶縁薬 | 感電（→麻痺の予防） | 感電を解除/軽減（麻痺の昇格を防ぐ） |

> ※状態異常の一覧・昇格（冷却→凍結、感電→麻痺）はステータス仕様に従う。

---

### 4.3 バフ（一定時間の強化）

| 名前 | 効果（概要） |
|---|---|
| 風走りの香 | 移動速度↑ |
| 守護の香 | 被ダメ軽減↑ |
| 剛力の香 | 与ダメ↑ |

> バフの持続時間・倍率は v1実装で決める（HUD上にバフアイコン表示）。

---

### 4.4 投擲（投げる道具）

| 名前 | 効果（概要） |
|---|---|
| 火炎瓶 | 小範囲に炎効果（燃焼付与 など） |
| 氷結瓶 | 小範囲に氷効果（冷却付与 など） |
| 雷瓶 | 小範囲に雷効果（感電付与 など） |
| 毒瓶 | 小範囲に毒効果（中毒付与 など） |

> 付与量・範囲・持続などは v1実装で決める（状態異常ルールはステータス仕様に従う）。

---

### 4.5 鑑定（未鑑定装備の鑑定）

| 名前 | 役割（概要） |
|---|---|
| 鑑定本 | 未鑑定装備を鑑定する |
| 上級鑑定本 | 鑑定本の上位版（※鑑定対象数/追加効果は後で確定） |
| 鑑定手帳 | 複数回使用できる鑑定アイテム（※回数は後で確定） |

> 宝箱から出る装備（武器・スキルチップ）は未鑑定になる前提。未鑑定のまま使用できるが呪いの可能性がある。

## 5. 装備（概要）

### 5.1 武器
- プレイヤーは武器を装備できる（初期武器枠=1、最大8本まで拡張）
- 武器は **WeaponDef（定義）** と **WeaponInstance（個体）** に分けて管理する
- 武器はスキルチップ（skills[]）とフォーメーション（formation_id）を持つ  
  ※詳細は武器システム仕様を参照

### 5.2 防具
- 防具は **1つのみ装備可能**
- 防具には **耐久値** があり、被ダメ等で減少し、0で破損
- 地上で修理可能／同一防具を合成して **+99** まで強化可能（詳細は別途）

### 5.3 アクセサリー
- 初期1枠、解放で最大3枠
- アクセサリーにはパッシブスキルが付く（レアで2個の可能性）
- 同一アクセサリーを合成して **+99** まで強化可能（詳細は別途）

---

## 6. スキル（概要）

### 6.1 スキルチップ
- 武器に装着する強化パーツ
- `skills[]` は基本的に `{ "id": "skill_id_xxx", "plus": n }` の配列で表現する
- パッシブは常時、攻撃系は武器の攻撃タイミングで発動する
- 複数属性が載っている場合は同時発動（炎＋毒＋氷など）

> スキルチップの種類（パッシブ/攻撃/軌道/複製/相性強化）と詳細パラメータは武器システム仕様を参照。

### 6.2 敵用スキルアイテム（概要）
- 敵は「敵専用スキルを持つアイテム」を落とすことがある
- 武器合成の“素材”として使える（ベースにはできない）
- 敵からしか得られないスキルが存在する

## 7. データ例（最小）

### 7.1 ItemDef（例：薬草）
```json
{
  "id": "item_herb_01",
  "name_key": "name_item_herb_01",
  "description_key": "desc_item_herb_01",
  "icon_file_name": "item_herb_01.png",

  "category": "consumable",
  "sub_type": "heal",
  "max_stack": 20,
  "usable_in_quickslot": true,

  "use_params": {
    "heal_tier": "small"
  }
}
```

### 7.2 ItemDef（例：風走りの香）
```json
{
  "id": "item_buff_wind_01",
  "name_key": "name_item_buff_wind_01",
  "description_key": "desc_item_buff_wind_01",
  "icon_file_name": "item_buff_wind_01.png",

  "category": "consumable",
  "sub_type": "buff",
  "max_stack": 10,
  "usable_in_quickslot": true,

  "use_params": {
    "buff_id": "buff_movespeed"
  }
}
```

### 7.3 ItemDef（例：鑑定本）
```json
{
  "id": "item_appraise_book_01",
  "name_key": "name_item_appraise_book_01",
  "description_key": "desc_item_appraise_book_01",
  "icon_file_name": "item_appraise_book_01.png",

  "category": "appraise",
  "sub_type": "identify",
  "max_stack": 10,
  "usable_in_quickslot": false,

  "use_params": {
    "identify_count": 1
  }
}
```
