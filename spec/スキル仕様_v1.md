## 0. スコープ

本書で定義するもの：
- **スキルチップ**：武器に装着する強化・効果付与パーツ全般
- **スキルチェーン**：スキルの発動順序と連鎖ルール
- **SkillDef / SkillInstance**：スキルのデータ構造（マスターデータと個体）
- **チップ分類**：Passive / Modifier / Attack / 軌道 / 複製 / 相性強化
- **軌道チップ**：フォーメーション/軌道パラメータを改造するスキル
- **複製チップ**：武器の本数を増加させるスキル
- **plus（+）のスケーリング**：スキル強化値による効果量の増幅ルール
- **敵用スキルアイテム**：敵がドロップするスキル素材

本書で定義しないもの（別仕様）：
- 武器データ構造（WeaponDef / WeaponInstance）・フォーメーション定義・攻撃周期：武器システム仕様を参照
- 状態異常の付与量・スタック・DoT計算：ステータス仕様を参照
- アイテム全般（消耗品・装備・鑑定）：アイテム・装備・スキル仕様を参照

## 1. 用語

- **スキルチップ（SkillChip）**：武器に装着する強化パーツ。`id` と `plus`（強化+）で表現する。
- **SkillDef（スキル定義）**：スキルのマスターデータ。IDごとに固定の情報（名前、説明、効果種別、パラメータ等）。
- **SkillInstance（スキル個体）**：武器に刺さっている個体。`{ "id": "skill_id_xxx", "plus": n }` で表現。
- **スキルチェーン**：`skills[]` の左→右に流れるスキル実行順序。攻撃が当たったら次の攻撃が起動する仕組み。
- **Passive（パッシブ）**：どこに置いても常時影響するスキル（順序無関係）。
- **Modifier（付与/改造）**：その位置以降の攻撃に付与されるスキル（順序あり）。
- **Attack（攻撃）**：命中判定を持つ攻撃アクション（順序あり）。
- **軌道改造チップ**：フォーメーション/軌道パラメータを改造するスキル（装着中は常時効果）。
- **複製チップ**：武器オブジェクトの本数を増やすスキル（装着中は常時効果）。

## 2. スキルチップ概要

### 2.1 基本ルール

- 武器に装着する強化パーツ。
- `skills[]` は基本的に `{ "id": "skill_id_xxx", "plus": n }` の配列で表現する。
- スキル発動は **左→右のスキルチェーン**（順序が意味を持つ）。
- 武器に乗っているスキルは全て発動する。
- 武器に乗っているパッシブスキルは常に発動する（攻撃力+何パーセントなど）。
- 武器に乗っている攻撃スキルは攻撃時に発動する。
- 武器に複数の属性が載っているときは攻撃時に全て発動する（毒と氷と炎が載っていたら全てが同時に発動する）。

### 2.2 敵用スキルアイテム

- 敵は「敵専用スキルを持つアイテム」を落とすことがある。
- 武器合成の"素材"として使える（ベースにはできない）。
- 敵からしか得られないスキルが存在する。

### 2.3 敵の武器のスキルチェーン適用
- 敵の武器でもプレイヤーと同様にスキルチェーンを適用する。
- `projectile.move_direction = "to_target"` は敵側ではプレイヤーをターゲットとして解決する。
- 敵チェーン由来の状態異常（例：poison）はプレイヤーにも適用される。
- 敵武器チェーンの起動条件（`start_spawn_timing` / `chain_trigger` / 安全柵 `MAX_CHAIN_SPAWN`）はプレイヤー武器と同一ルールとする。

## 3. チップ分類（v1.4.2）

スキルチェーン導入により、攻撃系チップを **「Attack」と「Modifier」** に分ける。

- **Passive（パッシブ）**：常時発動（位置無関係）
  - 例：攻撃力+%、クリ率+%、状態異常耐性+、移動速度+ など
- **Modifier（付与/改造）**：右側（以降）の攻撃に付与（位置依存）
  - 例：毒付与、炎付与、範囲+、ノックバック+、弾速+、状態付与量+ など
- **Attack（攻撃）**：命中判定を持つ攻撃アクション（位置依存）
  - 例：飛行弾、爆発、連鎖雷、設置床、斬撃波 など
- **軌道改造チップ**：フォーメーション/軌道パラメータを改造（装着中は常時効果）
  - 例：回転半径+、回転速度+、位相ずれ+、貫通回数+、ヒット回数+ など
- **複製チップ**：本数増加（装着中は常時効果）
- **相性強化チップ（任意）**：特定リアクションを伸ばす（例：爆燃の範囲+）

> スキルチェーン処理では「軌道/複製/相性強化」は Passive と同様に"常時ON"として扱ってよい。

---

## 4. タグ設計（相性の前提）

各チップは最低限以下のタグを持つ。

- **種別タグ**：`skill_type:passive | modifier | attack | orbit | replicate | reaction_boost`（運用/フィルタ/デバッグ用）

- **属性タグ**：物理 / 炎 / 氷 / 毒 / 雷
- **付与状態タグ**：燃焼、中毒…など（付与する状態異常）

---

## 5. スキルチェーン（左→右）発動ルール（v1.4.2）

- スキル実行は **skills[] の左→右** に流れる（順序が意味を持つ）。
- 同時発動はしない。**「左の攻撃が当たったら、右の攻撃が起動する」** を基本とする。

### 5.1 スキルの扱い（最低3種）
- **Passive（パッシブ）**：どこに置いても常時有効（順序の影響なし）
  - 例：攻撃力+%、クリ率+%、移動速度+、各種耐性 など
- **Modifier（付与/改造）**：設置したModifierチップより右側の攻撃全てに効果を付与する（順序の影響あり）
  - 例：毒付与、炎付与、範囲+、ノックバック+、弾速+、貫通+ など
- **Attack（攻撃）**：命中判定を持つ攻撃アクション
  - 例：飛行弾、爆発、連鎖雷、設置床、斬撃波 など
  - Attack は **スキル自身に base_damage（基礎攻撃力）** を持つ（詳細は 5.6）

> 補足：**軌道チップ**・**複製チップ**は「装着中は常時効果」を持つため、実装上は Passive と同様に扱ってよい（順序不問）。

### 5.2 1回の武器攻撃での開始点（Entry Attack）
武器のクールダウンが回ったとき、以下のいずれかを「開始攻撃」として実行する。

- skills[] 内に Attack が **1つでもある**：**最も左の Attack** が開始攻撃になる  
  - 通常攻撃も同時に発生する
- skills[] 内に Attack が **無い**：通常の **武器接触攻撃（ベース攻撃）** を開始攻撃として扱う

### 5.3 攻撃→攻撃の連鎖（OnHit起動）
- Attack が敵に **命中したとき**、その Attack の **右側** にある「次の Attack」を **1段** 起動する。
- 起動した Attack も命中すれば、さらに右側の次 Attack を起動できる（チェーン）。
- Attack の右側に Attack が存在しない場合、チェーンは終了。

> 連鎖は **命中イベント** 単位で処理する（乱数・状態付与・リアクション判定も命中イベント単位）。  
> 例：`飛行弾 → 爆発` は「飛行弾の命中イベント」の後に「爆発の命中イベント」が発生する。

### 5.4 Modifier の付与範囲（"その位置以降"）
- Modifier は、**その Modifier より右側** の Attack（およびベース攻撃）に付与される。
- Modifier が **Attack より左** にある場合、その Attack に効果が乗る。
- Modifier が **Attack より右** にある場合、その Attack には効果が乗らない（後段にのみ乗る）。

例：
- `飛行弾 → 爆発`：飛行弾命中で爆発起動
- `飛行弾 → 毒 → 爆発`：飛行弾には毒なし／爆発には毒付与
- `毒 → 飛行弾 → 爆発`：飛行弾・爆発の両方に毒付与
- `毒（先頭）` ＋ Attack無し：通常の武器接触攻撃（ベース攻撃）に毒付与

### 5.5 Passive/orbit の扱い（位置無関係）
- Passive/orbit は skills[] の **どこにあっても常時影響**する（チェーン開始点や付与範囲に依存しない）。

### 5.6 攻撃スキルのダメージ（base_damage）
- Attack は `base_damage` を持ち、ダメージ計算は「ステータス仕様」の順序に従う。
- **武器接触攻撃（ベース攻撃）** の場合は `WeaponDef.base_damage` を `base_damage` として扱う。
- チップ倍率・パッシブ倍率などは、Attack/ベース攻撃の区別なく同じルールで乗算する。
- ダメージ計算は「攻撃者側ステータス」を使う（プレイヤー時は player、敵チェーン時は enemy）。
- 敵チェーン時の Attack は `SkillDef.base_damage` を起点に、`enemy.damageMult` と `enemy.attackScale` を乗算して計算する（Modifier由来倍率も同時に乗算）。

### 5.7 暴れ防止（推奨の安全柵）
チェーンは「多段ヒット」「貫通」「範囲攻撃」と組み合わさると起動回数が爆発しやすい。v1では以下を推奨。

- **1命中イベントにつき、次段Attackは1回だけ起動**（多段 `hit_num` で増やさない）
- **1攻撃インスタンスあたりの最大起動数 cap**（例：`MAX_CHAIN_SPAWN = 16`）を用意する

## 6. SkillDef（スキルチップ定義 / マスターデータ）

武器の `skills[]` は **SkillInstance** の配列で、各要素は **SkillDef**（マスターデータ）を参照する。

- **SkillInstance（武器に刺さっている個体）**
  - `{ "id": "skill_id_xxx", "plus": n }`
  - `id`：参照する SkillDef のID
  - `plus`：そのスキルの強化値（0〜max_plus）。効果量の増幅に使う（6.2）

- **SkillDef（スキル定義）**
  - 例：`db/skill_db/skill_id_xxx.json`
  - 目的：表示（name/desc/icon）と、効果（攻撃/付与/パッシブ/軌道/複製…）を一元管理する

---

### 6.1 SkillDef：共通フィールド（v1 最低限）

```json
{
  "id": "skill_id_poison01",
  "name_key": "name_skill_poison01",
  "description_key": "desc_skill_poison01",

  "skill_type": "modifier",
  "rarity": "common",

  "max_plus": 99,
  "unique_per_weapon": false,

  "tags": [
    "skill_type:modifier",
    "element:poison",
    "ailment:poison"
  ],

  "ui": {
    "icon_file_name": "skill_poison_01.png",
    "sort_order": 200
  },

  "params": {}
}
```

#### 各パラメータ説明

- `id`（string / 必須）
  - スキルの一意ID。`skill_id_...` 推奨。
  - Weaponの `skills[].id` から参照されるキー。

- `name_key`（string / 必須）
  - 表示名のローカライズキー。
  - 例：日本語テキストDBで `name_key → "毒付与"` のように解決。

- `description_key`（string / 必須）
  - 説明文のローカライズキー。

- `skill_type`（string / 必須）
  - スキルの種別。処理・UI分類の最上位。
  - 値：`passive | modifier | attack | orbit | replicate | reaction_boost`
    - `passive`：常時有効（位置無関係）
    - `modifier`：右側（以降）の攻撃に付与（位置依存）
    - `attack`：命中判定を持つ攻撃アクション（位置依存）
    - `orbit`：フォーメーション/軌道パラメータ改造（常時）
    - `replicate`：本数増加（常時）
    - `reaction_boost`：リアクション強化（常時、将来枠）

- `rarity`（string / 必須）
  - 抽選・ドロップ・UI色分けなどに使うレア度。
  - 例：`common | rare | legendary`（運用に合わせて）

- `max_plus`（int / 必須）
  - `plus` の上限。`0〜max_plus` の範囲で運用する。
  - 例：複製は `3`、一般スキルは `99` など。

- `unique_per_weapon`（bool / 必須）
  - 同一武器に同一 SkillDef を複数挿せるか。
  - `true`：同一武器内で「このスキルは1枚まで」（複製などに推奨）
  - `false`：複数枚OK（毒付与を2枚挿す等が可能）

- `tags`（string[] / 必須）
  - 抽選/フィルタ/デバッグ用の分類ラベル。
  - 推奨：最低限 `skill_type:*`、要素 `element:*`、状態 `ailment:*` を付ける。
  - 例：
    - `skill_type:modifier`：種別
    - `element:poison`：属性（攻撃/付与の入力としても使える）
    - `ailment:poison`：付与状態（中毒など）

- `ui`（object / 必須）
  - UI表示専用メタデータ（戦闘計算には影響しない）
  - `ui.icon_file_name`（string / 必須）
    - アイコン画像ファイル名（例：`skill_poison_01.png`）
  - `ui.sort_order`（int / 必須）
    - 一覧表示順（小さいほど先に表示）。フィルタ/並び替えで使用。

- `params`（object / 必須）
  - スキル本体の挙動パラメータ。
  - 中身は `skill_type` ごとに変わる（6.3）。

---

### 6.2 plus（+）の基本スケーリング（v1 推奨）

SkillInstance の `plus` は SkillDef の効果量を増幅する。  
**「何を増やすか」**はスキル種別で変わる（ダメージ/付与/パッシブ等）。

- **状態異常付与量（apply）**
  - 付与量はステータス仕様の `ApplyMult(plus)`（逓減など）を使う。
  - 例（概念式）：
    - `apply = ApplyBase * ApplyMult(plus) * (1 + ARC * A_APPLY) * ...`
  - 用語：
    - `ApplyBase`：スキル定義が持つ基礎付与量（例：0.60）
    - `ApplyMult(plus)`：+に応じた倍率（逓減式推奨）
    - `ARC`：主人公ステ（相性/付与系の伸び）
    - `A_APPLY`：ARCの係数（定数）

- **攻撃の基礎ダメージ（base_damage）**
  - 攻撃スキルが持つ `base_damage` を起点に、+で強化する。
  - 例（概念式）：
    - `SourceBaseDamage = base_damage * DamagePlusMult(plus)`
  - 用語：
    - `base_damage`：スキル自体の基礎攻撃力
    - `DamagePlusMult(plus)`：+の倍率（武器強化と同系の逓減式推奨）

- **パッシブ（%や固定値）**
  - v1はまず線形で開始し、暴れたら逓減に切替できる形にする。
  - `value = base + per_plus * plus`
  - 例：攻撃力+5%（0.05）に、+ごとに+1%（0.01）など

> 注：スキルごとに「線形/逓減」を切り替えたい場合は、各 `params` の中に `scaling` を追加してもよい  
> （例：`{ "scaling": "linear" }` / `{ "scaling": "diminishing", "half": 25, "max": 2.0 }` など）

### 6.3 skill_type ごとの params 最低限（説明つき）

#### A) passive（常時有効・位置無関係）

```json
"params": {
  "mods": [
    { "stat": "ChipDamagePct", "op": "add", "base": 0.05, "per_plus": 0.01 }
  ]
}
```

- `mods`（array / 必須）
  - 複数の常時効果を並べられるようにする配列。

各 `mods[]` 要素：

- `stat`（string / 必須）
  - どのステータス（内部バフ枠）を変更するか。
  - v1 推奨：
    - `ChipDamagePct`：ダメージ計算の「チップ等の倍率」枠に加算（0.05 = +5%）
    - `MoveBonusPct`：移動速度に加算（0.10 = +10% など）
    - `CritBonus`：クリ率に加算（0.05 = +5%pt など）
    - `CritDmgBonus`：クリ倍率に加算（0.20 = +20% など）
    - `AilmentTakenMult`：状態異常の受け/効き（将来枠として）

- `op`（string / 必須）
  - 演算種別（v1は最小で `add` のみでOK）
  - `add`：最終値に加算（例：+5%）

- `base`（number / 必須）
  - `plus=0` のときの基礎値。
  - `%` 系は **小数（0.05 = 5%）**で表現する。

- `per_plus`（number / 必須）
  - `plus` 1あたりの増加分。
  - 例：0.01 なら +1% ずつ伸びる。

（任意）`cap_min/cap_max` を後で足すと暴れ防止がしやすい。

---

#### B) modifier（右側の攻撃に付与・位置依存）

```json
"params": {
  "add_tags": ["element:poison", "ailment:poison"],
  "apply_ailments": [
    { "ailment_id": "poison", "apply_base": 0.60 }
  ],
  "add_attack_damage_pct": 0.00
}
```

- `add_tags`（string[] / 任意）
  - **このModifier以降の攻撃**に追加されるタグ。
  - 例：後段Attackを `element:poison` 扱いにする／デバッグ可視化に使う。

- `apply_ailments`（array / 任意）
  - 命中時に付与する状態異常のリスト（複数付与も可能）。
  - 要素：
    - `ailment_id`（string / 必須）
      - 付与する状態異常ID（例：`poison`, `burn` など）
    - `apply_base`（number / 必須）
      - 基礎付与量（端数方式で蓄積する前提の float）
      - 例：`0.60` は「毎ヒット 0.6 スタック相当を足す」イメージ
      - 実効付与は 6.2 の式（`ApplyMult(plus)` など）で増える

- `add_attack_damage_pct`（number / 任意 / 既定 0）
  - **このModifier以降の攻撃**に加算されるダメージ倍率。
  - 0.10 = +10%（主にビルドの微調整用。多用しないのが安全）

---

#### C) attack（命中判定を持つ攻撃アクション・位置依存）

```json
"params": {
  "attack_kind": "projectile",
  "base_damage": 10,

  "damage_element": "physical",
  "hit": { "hit_num": 1, "pierce_count": 0 },

  "start_spawn_timing": "start",
  "chain_trigger": "on_hit",

  "projectile": {
    "speed_tile_per_sec": 10.0,
    "life_sec": 1.2,
    "move_direction": "to_target",
    "sprite_effect_id": "effect_id_proj_basic_01",
    "hit_box_per":0.8,
    "is_rotate":true,
    "disappear_hit_wall":false
  },

  "se_key_fire": "se_shot_01",
  "se_key_hit": "se_hit_01",
  "effect_id_fire": "fx_shot_01",
  "effect_id_hit": "fx_hit_01"
}
```

- `attack_kind`（string / 必須）
  - 攻撃の形。v1最小：`projectile | aoe`
    - `projectile`：弾（移動体）を飛ばす
    - `aoe`：範囲（爆発など）をその場で発生
  - 拡張案（実装予定）：
    - `charge`：攻撃者本体が突進して命中判定を持つ
    - `summon`：敵/オブジェクトを召喚する
  - 使い分け（実装者向け）：
    - `projectile` は「弾オブジェクトが移動する攻撃」
    - `aoe` は「トリガー座標で範囲発生する攻撃」

- `base_damage`（number / 必須）
  - このAttack自体の基礎攻撃力（= ダメージ計算の起点）
  - 最終ダメージは主人公ステ・チップ・乱数・クリ等で増減（ステータス仕様に従う）。

- `damage_element`（string / 必須）
  - このAttackの属性。
  - 例：`physical | fire | ice | poison | thunder`
  - リアクション（相性）判定の入力にもなる。

- `hit`（object / 任意）
  - 多段/貫通など「命中のしかた」。
  - `hit.hit_num`（int / 任意 / 既定 1）
    - 同一敵への多段回数（通常 1）
  - `hit.pierce_count`（int / 任意 / 既定 0）
    - 追加で当てられる敵数（0なら最大1体、2なら最大3体）

- `start_spawn_timing`（string / 任意 / 既定 attacker）
  - 最初の攻撃スキルの発動タイミング
  - `start`：武器攻撃開始時にプレイヤー（武器中心）基準で生成
  - `hit`：武器攻撃がヒットヒットした命中座標から生成

- `chain_trigger`（string / 任意 / 既定 on_hit）
  - 次のAttackを起動する条件。v1は `on_hit` 固定でOK。
  - 将来：`on_fire`（発射時）や `on_kill`（撃破時）など拡張用に予約。

- `projectile`（object / attack_kind=projectile のとき必須）
  - `speed_tile_per_sec`（number / 必須）
    - 弾速（タイル/秒）。タイル=32px前提なら表示/当たりが整えやすい。
  - `life_sec`（number / 必須）
    - 弾の寿命（秒）。寿命切れで消滅（ヒットしなくても消える）。
  - `move_direction`(string/任意)
    - 移動方向。デフォルトでは一番近くの敵
    - `to_target`:projectile発生時に一番近くの敵に向かって移動。デフォルト設定
    - `guided`:projectile発生時にターゲットにした敵を追いかける誘導弾
    - `random`:ランダムの方向に向かって移動。
  - `sprite_effect_id`(string/必須)
    - スキルの描画はeffect_idを用いて行う。
    - 命中判定があるエフェクトを作る
    - 紐づくeffectの `loop=false` が終了してエフェクトが消滅したフレームでは、projectile本体（当たり判定）も同時に消滅する。
    - `loop=true` のeffectはエフェクト寿命連動で強制消滅せず、`life_sec` / 壁命中 / 命中上限で消滅判定する。
  - `hit_box_per`(float/任意)
    - sprite_effect_idで指定したeffectサイズのどれくらいの割合を命中判定領域とするか。
    - 0.5の場合は、全体の50%が当たり判定となる(100x100のエフェクトなら50x50)。なお中心はeffectの中心座標である。
    - 省略時はeffectの全体が当たり判定である
  - `is_rotate` (bool / 任意)
    - エフェクトを移動方向に応じて向きを変えるか
    - 省略時はtrue (向きを変える)
  - `disappear_hit_wall`（bool / 任意）
    - 弾が壁に当たった時に消えるか。デフォルトはtrue(消える)

- `aoe`（object / attack_kind=aoe のとき必須）
  - `sprite_effect_id`(string/必須)
    - スキルの描画はeffect_idを用いて行う。
    - **当たり判定はeffectのサイズである(命中判定があるエフェクトを作る)**
    - 当たり判定はエフェクト消滅とともに消える
  - `hit_box_per`(float/任意)
    - sprite_effect_idで指定したeffectサイズのどれくらいの割合を命中判定領域とするか。
    - 0.5の場合は、全体の50%が当たり判定となる(100x100のエフェクトなら50x50)。なお中心はeffectの中心座標である。
    - 省略時はeffectの全体が当たり判定である
  - `hit_interval_sec`（number / 任意 / 既定 0.0）
    - 0.0：単発（爆発）
    - >0：設置床のように一定間隔でヒット（将来用。v1は0推奨）
  - `target_position`（string / 任意 / 既定 `"hit"`）
    - AoEを発生させる座標の決定方法。
    - `hit`：スキルチェーンの前段が命中した座標に発生する（既存動作・プレイヤー武器の爆発等）
    - `attacker`：攻撃者自身の位置に発生する（自己中心の爆発・バフ等）
    - `target_locked`：**ターゲット（通常はプレイヤー）の位置** に発生する。`position_lock_timing` で座標ロックのタイミングを決める。敵専用AoE（ボスの石槌プレス等）で使用する
    - `target_locked` のAoEは、**`sprite_effect_id` の表示中心をAoE中心（テレグラフ中心）と一致**させる。
  - `position_lock_timing`（string / 任意 / 既定 `"on_fire"`）
    - `target_position="target_locked"` のとき有効。AoE座標を確定するタイミング。
    - `on_fire`：スキル発動時のターゲット位置を使う（effect表示中心もこの座標）。
    - `on_windup_start`：AI windup 開始時のターゲット位置をロックし、以後更新しない（ボスのプレス攻撃等で「予告位置に固定」する用途。effect表示中心もロック座標）。
  - `telegraph_style`（string / 任意）
    - AoE着弾予告の描画スタイル。effect画像は使わず、コード側のプリミティブ描画で表示する。
    - `circle_red_translucent`：半透明赤円（ボスの石槌プレス等の着弾範囲予告に推奨）
    - 省略時はテレグラフ表示なし（プレイヤー武器のAoE等）
  - `telegraph_radius_tiles`（number / 任意）
    - テレグラフ円の半径（タイル）。`telegraph_style` 指定時に有効。ボスキャラ仕様では 1.5〜2.0 タイルを推奨

- `charge`（object / attack_kind=charge のとき必須 / 拡張案）

  > **共通フィールドとの関係**：
  > - `base_damage`：突進命中時の衝突ダメージとして使用する。
  > - `damage_element`：突進の属性（通常は `physical`）。
  > - `hit.pierce_count`：突進経路上で巻き込める追加敵数。既定 0（最初の1体のみ）。`stop_on_player_hit=true` の場合、プレイヤー接触で停止するため pierce は実質無効。
  > - `start_spawn_timing`：charge では非該当（本体が移動するため）。省略してよい。
  > - `chain_trigger`：突進命中後に次段 Attack をチェーン起動する場合は `on_hit` を指定。

  - `dash_distance_tiles`（number / 必須）
    - 突進距離（タイル）。例：6.0〜8.0
  - `speed_tile_per_sec`（number / 必須）
    - 突進速度（タイル/秒）。通常移動の 2.5〜3.5 倍を目安にする
  - `direction_lock_timing`（string / 必須）
    - 突進方向の確定タイミング。
    - `on_windup_start`：windup 開始時にターゲット方向をロックし、以後更新しない（v1推奨・唯一の実装値）
    - 将来拡張：`on_windup_end`（windup 終了直前にロック）なども検討可
  - `micro_correct_deg`（number / 任意 / 既定 0）
    - windup終盤にターゲット方向へ補正する最大角度（°）。0＝補正なし。
    - **AIプロファイルの `phases[].charge_micro_correct_deg` が設定されている場合、フェーズ側の値で上書きされる**（スキル側はデフォルト値として機能する）。
  - `stop_on_player_hit`（bool / 任意 / 既定 true）
    - プレイヤー接触時に突進を停止するか
  - `wall_hit_recover_sec`（number / 任意）
    - 壁/柱に衝突したときのrecover秒数（回避成功報酬の調整用）。
    - AIプロファイルの `actions.charge.recover_on_wall_hit_sec` と同義。AI側で設定した場合はAI側の値を優先する。
  - `hit_box_tiles`（number / 任意 / 既定：敵画像幅の 0.8 倍）
    - 突進中の当たり判定の幅（タイル）。突進は弾ではなく本体移動のため、`sprite_effect_id` / `hit_box_per` の代わりにこのフィールドで当たり判定サイズを指定する。
    - 省略時は敵本体の画像幅 × 0.8 を当たり判定として使用する。
  - `telegraph_style`（string / 必須）
    - テレグラフ描画スタイル。`line_red_translucent` を推奨。
    - effect画像は使わず、コード側のプリミティブ描画（半透明赤ライン）で表示する。
  - `telegraph_width_tiles`（number / 任意 / 既定 1.0）
    - テレグラフ（予告ライン）の表示幅（タイル）。ボスキャラ仕様では 0.8〜1.0 タイルを推奨。
  - `knockback_tiles`（number / 任意 / 既定 0）
    - 命中時のノックバック距離（タイル）。0＝ノックバックなし。

- `summon`（object / attack_kind=summon のとき必須 / 拡張案）

  > **共通フィールドとの関係**：
  > - `base_damage`：召喚行動自体はダメージを与えない。`0` を設定するか省略する。
  > - `damage_element`：非該当。省略してよい。
  > - `hit`：非該当。省略してよい。
  > - `start_spawn_timing`：非該当。省略してよい。
  > - `chain_trigger`：召喚完了後に次段 Attack をチェーン起動したい場合は `on_hit` を指定できるが、通常は不要。

  - `enemy_id`（string / 必須）
    - 召喚する enemy_db の ID。例：`"OgreMinion_01"`
  - `count`（object / 必須）
    - 1回の召喚で出す体数。
    - `count.min`（int / 必須）：最小召喚数
    - `count.max`（int / 必須）：最大召喚数（min〜max の範囲で乱数決定）
    - **AIプロファイルの `phases[].summon_count` が設定されている場合、フェーズ側の値で上書きされる**（スキル側はデフォルト値として機能する）。
  - `spawn_style`（string / 必須）
    - 出現位置の決定方針。
    - `boss_ring_outside`：ボス周囲の一定距離外（プレイヤー足元直湧きを避ける）
    - `room_corners`：部屋の四隅付近
    - `near_pillars`：柱の裏側（テレグラフで視認できる配置）
    - 実装時にスタイルを追加可能。スタイルに該当する空きタイルがない場合はランダム空きタイルにフォールバックする。
  - `spawn_telegraph_sec`（number / 任意 / 既定 0.5）
    - 出現地点マーカーの予告時間（秒）。この時間だけ先にマーカーを表示し、経過後に召喚体が出現する。
  - `spawn_telegraph_style`（string / 任意）
    - 出現地点テレグラフの描画スタイル。`circle_red_translucent` を推奨。
    - effect画像は使わず、コード側のプリミティブ描画（半透明赤円）で表示する。
  - `spawn_telegraph_radius_tiles`（number / 任意 / 既定 0.5）
    - 出現地点テレグラフ円の半径（タイル）。
  - `max_alive_in_room`（int / 任意）
    - 部屋内に同時存在できる召喚体の上限。超えている場合は召喚しない。
    - AIプロファイルの `summon_rules.max_alive_in_room` と同義。AI側で設定した場合はAI側の値を優先する。
  - `max_alive_per_summoner`（int / 任意）
    - 召喚者1体あたりの同時存在上限。
    - AIプロファイルの `summon_rules.max_alive_per_summoner` と同義。AI側で設定した場合はAI側の値を優先する。
  - `vanish_on_summoner_death`（bool / 任意 / 既定 false）
    - 召喚者死亡時に全召喚体を即時消滅させるか（テンポ維持用）。
    - AIプロファイルの `summon_rules.vanish_on_summoner_death` と同義。AI側で設定した場合はAI側の値を優先する。
  - `cast_effect_id`（string / 任意）
    - 詠唱時に再生するeffect_id（咆哮演出など）。空/未設定なら再生しない。

- `se_key_fire`（string / 任意）
  - 攻撃を生成/発射した時のSEキー。空/未設定なら鳴らさない。

- `se_key_hit`（string / 任意）
  - 命中時のSEキー。空/未設定なら鳴らさない。

- `effect_id_fire`（string / 任意）
  - 生成/発射時に再生するエフェクトID。空/未設定なら再生しない。

- `effect_id_hit`（string / 任意）
  - 命中時に再生するエフェクトID。空/未設定なら再生しない。

- `per_plus`（number / 必須）

#### E) replicate（複製・常時）

```json
{
  "id": "skill_id_replicate01",
  "skill_type": "replicate",
  "rarity": "rare",
  "max_plus": 3,
  "unique_per_weapon": true,
  "tags": ["skill_type:replicate"],
  "ui": { "icon_file_name": "skill_replicate_01.png", "sort_order": 50 },
  "params": {
    "copies_base": 1,
    "copies_per_plus": 1
  }
}
```

- `params.copies_base`（int / 必須）
  - 生成本数の基礎値。
  - v1推奨：1（= もともと1本）

- `params.copies_per_plus`（int / 必須）
  - `plus` 1あたり増える本数。
  - v1推奨：1（= `1 + plus` 本と同等）

- 本数の計算
  - `n = copies_base + copies_per_plus * plus`
  - 例：`copies_base=1, copies_per_plus=1`
    - plus=0 → 1本
    - plus=1 → 2本
    - plus=3 → 4本

- `unique_per_weapon=true` 推奨
  - 1武器に複製は1枚まで（暴れ防止）。

## 7. 軌道パラメータと軌道チップ

### 7.1 代表的な軌道パラメータ
- **回転半径**：武器が回る距離
- **回転速度**：武器が回る速さ
- **位相ずれ**：複数本の配置間隔（密集/拡散）
- **貫通回数**：敵を貫く回数
- （攻撃）**ヒット回数（hit_num）**：同一敵への多段（通常1）

### 7.2 軌道チップ（例）
- 回転半径+
- 回転速度+
- 位相ずれ+（密集/拡散の方向性をチップで分けてもよい）
- 貫通回数+
- ヒット回数+（多段化）

### 7.3 体感の指針（見た目変化を重視）
- 数値上昇は、可能な限り**視覚的変化**（密度・広がり・残像など）を伴う。
- 本数増加（複製）と位相ずれは特に「増えた感」が出るため優先的に演出を入れる。

> フォーメーション（軌道）の定義（circle / arc / line 等）自体は **武器システム仕様_v1_4_1.md** のセクション4を参照。

## 8. 本数増加（複製チップ）

### 8.1 目的
- 武器1本につき n 本へ増やし、見た目の爽快感と面制圧力を上げる。
- ただし無制限のDPS増加を防ぐため、ルールで抑制する。

### 8.2 仕様
- 「複製」スキルチップを装着すると、その武器枠から生成される武器オブジェクト本数が増える。
- **1武器につき複製チップは1枚のみ装着可能**
- **複製の+上限は3**
- 生成本数：`n = 1 + 複製の+値`
  - 複製+1 → 2本
  - 複製+2 → 3本
  - 複製+3 → 4本
- 生成された複数本は、位相ずれに応じて配置される。

### 8.3 攻撃周期の共有（複製）
- 同一武器から生成された複数本（複製）は、**同じ攻撃周期（クールダウン）を共有**する（= `attack_seq` も同時に進む）。
- ヒット判定は **各コピーが独立**（同一敵にもそれぞれ当たる）を基本とし、総火力の暴れは **逓減補正（8.4）** で制御する。
  - ※もし単体相手の瞬間火力が強すぎる場合は、調整ノブとして「同一敵は武器枠ごとに1回まで（hit_setを武器枠で共有）」へ切替可能。

### 8.4 本数増加時の逓減補正（推奨初期式）
- 本数増加は総ダメージが逓減補正される。
- 推奨初期式（調整可能）：
  - 総ダメ倍率 `= 1 + 0.6×(n-1)`
  - 1本あたりの係数 `= 総ダメ倍率 / n`
  - 例：
    - 2本：総1.6倍（1本0.8倍）
    - 3本：総2.2倍（1本0.733倍）
    - 4本：総2.8倍（1本0.7倍）

## 10. チェーン実装のための解決手順（データ視点・最小）

- **Passive**：全スキルから集約（位置無視）
- **Attack i の解決**：`i` より左側にある **Modifier** をすべて合成して、Attackの `ResolvedAttack` を作る  
  - `add_tags` をマージ
  - `apply_ailments` をマージ
  - `add_attack_damage_pct` を加算
- **Attack の base_damage** は「スキル自身の base_damage」を起点に、攻撃者ステータス（POW等）と各倍率で計算する
- 敵チェーンでは `SkillDef.base_damage × enemy.damageMult × enemy.attackScale × (1 + add_attack_damage_pct合計)` を基準式とする
- **OnHit** で右側の「次のAttack」を起動（スキルチェーンの仕様 5 節に従う）
